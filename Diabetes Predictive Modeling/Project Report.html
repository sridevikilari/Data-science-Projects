Objective¶

The aim of this project is to predict whether an individual tests positive or negative for diabetes using Pima Indians Diabetes Database from OpenML. By fitting and comparing five different binary classifiers to the database, the identified best classifier will be proposed for prediction.

Methodology¶

Data Description¶

The Pima Indians Diabetes dataset has diagnostic information of 768 female patients who are at least 21 years old and of Pima Indian heritage. It has 8 predictor variables and 1 target feature which has two classes defined as tested_positive and tested_negative respectively. There are no missing values.

The dataset consists of following features :

  • preg : Number of times pregnant

  • plas : Plasma glucose concentration a 2 hours in an oral glucose tolerance test

  • pres : Diastolic blood pressure (mm Hg)

  • skin : Triceps skin fold thickness (mm)

  • insu : 2-Hour serum insulin (mu U/ml)

  • mass : Body mass index (weight in kg/(height in m)^2)

  • pedi : Diabetes pedigree function(a function which scores likelihood of diabetes based on family history)

  • age : Age (years)

  • class: Test result positive or negative

K-Nearest Neighbors (KNN)¶¶

For KNN k-neighbhors and distance metric(P) are hyperparameters that need to be tuned and for feature selection we define a function RFIFeatureSelector() using BaseEstimator, TransformerMixin from sklearn

In [34]:
from sklearn.base import BaseEstimator, TransformerMixin

# custom function for RFI feature selection inside a pipeline
# here we use n_estimators=100
class RFIFeatureSelector(BaseEstimator, TransformerMixin):
    
    # class constructor 
    # make sure class attributes end with a "_"
    # per scikit-learn convention to avoid errors
    def __init__(self, n_features_= 8):
        self.n_features_ = n_features_
        self.fs_indices_ = None

    # override the fit function
    def fit(self, X, y):
        from sklearn.ensemble import RandomForestClassifier
        from numpy import argsort
        model_rfi = RandomForestClassifier(n_estimators=100)
        model_rfi.fit(X, y)
        self.fs_indices_ = argsort(model_rfi.feature_importances_)[::-1][0:self.n_features_] 
        return self 
    
    # override the transform function
    def transform(self, X, y=None):
        return X[:, self.fs_indices_]

# custom function to format the search results as a Pandas data frame
def get_search_results(gs):

    def model_result(scores, params):
        scores = {'mean_score': np.mean(scores),
             'std_score': np.std(scores),
             'min_score': np.min(scores),
             'max_score': np.max(scores)}
        return pd.Series({**params,**scores})

    models = []
    scores = []

    for i in range(gs.n_splits_):
        key = f"split{i}_test_score"
        r = gs.cv_results_[key]        
        scores.append(r.reshape(-1,1))

    all_scores = np.hstack(scores)
    for p, s in zip(gs.cv_results_['params'], all_scores):
        models.append((model_result(s, p)))

    pipe_results = pd.concat(models, axis=1).T.sort_values(['mean_score'], ascending=False)

    columns_first = ['mean_score', 'std_score', 'max_score', 'min_score']
    columns = columns_first + [c for c in pipe_results.columns if c not in columns_first]

    return pipe_results[columns]

Limitations and Proposed Solutions¶

Due to possible data leakage, all features from the dataset are considered without undergoing feature reduction. Example: The higher value in Plasma glucose concentration test (plas) indicates possibility of diabetes. In future, we could perform more detailed analysis on feature selection & ranking process.

Secondly, in our data set we observed class imbalance problem and handled it by splitting the dataset using random state and skipped sampling as the number of instances are less than 1000.

In future, for class imbalance we need to do under sampling which means samples should be taken in equal ratio. The accuracy can be improved by doing under sampling. Further more, parameter tuning can be done for Logistic regression models.

Summary¶

From SVM model produces the highest cross-validated AUC score on the training data followed by Naive Bayes. And we received similar output while evaluating on test data.

The SVM model and Logistic regression model had more eeighted average recall compared to the rest. Our models are not sensitive to the number of features as selected by RFI when no.of features combinations were changes on KNN, Naive-Bayes. Hence, we have applied our original 8 features on our models, which avoided overfitting and resulted in models that were easier to train.
